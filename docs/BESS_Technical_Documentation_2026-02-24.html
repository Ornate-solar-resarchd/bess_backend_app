5. Lifecycle and Validation Rules

5.1 Shipment status guardrails

  • Transition to PACKED requires assigned units count to meet expected_quantity.
  • Transition to PACKED also requires at least one uploaded shipment document.
  • Status transitions propagate BESS stage updates according to mapping in shipment service logic.

5.2 File upload validations

  • Multipart field file is mandatory for shipment and upload APIs.
  • Empty file payloads are rejected.
  • Checklist photo upload restricts to image content types.
  • Document uploads and shipment document uploads persist generated filenames and return URLs.

9. Glossary of Common Terms

TermMeaning in this system
BESS Unit A single battery energy storage unit tracked from registration through shipment, installation, and commissioning.
Checklist Structured list of tasks and validations that must be completed at each stage (e.g., installation, inspection) with optional photo proof.
Shipment A logical grouping of BESS units moving together from one location to another (warehouse to site, site to RMA, etc.).
Stage The lifecycle position of a unit (e.g., REGISTERED → IN_TRANSIT → INSTALLED → COMMISSIONED).
Assignment A task or site visit assigned to an engineer, with its own accept/decline/complete lifecycle.
Commissioning Record A detailed record of technical checks and measurements captured when a BESS unit is formally commissioned.
Media Storage Filesystem path where all uploaded photos/documents are written, exposed read-only via the /media mount.

10. Example Journey: From Registration to Commissioning

This example illustrates how the main building blocks work together for a single BESS unit.

  • Step 1 – Register: A unit is registered via /api/v1/bess/ or QR/photo-based registration. Core identifiers and model details are stored.
  • Step 2 – Plan shipment: A shipment is created via /api/v1/shipments/ with an expected_quantity. The unit is assigned to the shipment (possibly via bulk assignment).
  • Step 3 – Guardrails before PACKED: The backend enforces that assigned units count matches expected_quantity and that at least one shipment document is uploaded before status moves to PACKED.
  • Step 4 – Transit and arrival: Shipment status transitions trigger mapped stage changes on each BESS unit, keeping unit lifecycle in sync.
  • Step 5 – Site checklist: Engineers retrieve installation checklists, update items, and attach photo evidence through checklist and upload APIs.
  • Step 6 – Commissioning: After installation tasks pass validation, commissioning records are created via /commissioning/{bess_unit_id}/records, closing the lifecycle.
